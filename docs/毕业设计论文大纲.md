# 餐饮个性化推荐系统——毕业设计论文大纲

> 依据《毕业设计论文规范.md》撰写，以中文菜品业务线为主线，Yelp 数据集作为推荐算法实验与大数据处理的底层数据载体。

---

## 摘要

**中文摘要：**
随着互联网餐饮平台的快速发展，用户面临的信息过载问题日益严重。本文设计并实现了一个基于 Django 的餐饮个性化推荐系统，旨在通过推荐算法帮助用户快速发现感兴趣的菜品。系统以中文菜品业务为核心，支持用户注册登录、菜品浏览、收藏、评论以及个性化推荐等功能。在推荐算法层面，系统集成了基于收藏的协同过滤推荐（ItemCF / UserCF），并引入 TF-IDF 内容推荐与 Spark ALS 矩阵分解作为算法实验与大数据能力验证。工程实现上，系统采用"离线计算、在线轻量读取"的策略，将相似度构建、模型训练等重计算移出请求链路，保证了页面响应速度与演示稳定性。最后，通过功能测试验证了系统的可用性与降级策略的鲁棒性。

**关键词：** 个性化推荐；协同过滤；TF-IDF；Spark ALS；Django；餐饮推荐

## 第一章 绪论

### 1.1 研究背景
- 互联网餐饮信息爆炸，用户在海量菜品中做决策的成本不断上升。
- 个性化推荐技术已成为缓解信息过载、提升用户体验的重要手段。
- 将推荐算法从理论研究落地为可运行、可演示的工程系统，具有较高的实践价值。

### 1.2 研究意义
- **理论意义：** 将协同过滤、内容推荐、矩阵分解等算法在一个统一系统中进行工程化落地，探索中小规模系统下算法复杂度与工程可演示性的平衡。
- **实际意义：** 构建一套完整的餐饮推荐系统，为后续推荐算法研究提供基础平台，同时可作为毕业设计进行完整的功能与算法演示。

### 1.3 国内外研究现状
- 协同过滤推荐：基于用户行为相似性或物品相似性进行推荐，是推荐系统中最经典的方法之一<sup>[1]</sup>。
- 内容推荐：通过提取物品文本特征（如 TF-IDF）计算内容相似度，常用于新物品推荐和冷启动场景<sup>[2]</sup>。
- 矩阵分解与大数据推荐：ALS（交替最小二乘法）等隐语义模型在处理大规模稀疏数据上表现优异，Spark 等分布式计算框架为大规模离线训练提供了工程支撑<sup>[3]</sup>。
- **现有不足与本项目切入点：** 现有研究多聚焦于算法本身的精度提升，而如何在有限时间和资源约束下，将多种推荐算法集成到一个稳定、可演示的 Web 系统中，仍有较强的工程实践需求。本项目即以此为目标，构建一个功能闭环、算法可解释、工程可控的餐饮推荐系统。

## 第二章 相关技术介绍

### 2.1 Django Web 框架
- **作用：** 系统后端主框架，负责业务逻辑、页面渲染、数据库 ORM 操作以及管理命令封装。
- **特点：** 采用 MTV（Model-Template-View）架构，内置用户认证、中间件、管理后台，适合快速构建内容型网站。

### 2.2 MySQL 数据库
- **作用：** 作为业务数据持久化层，存储用户、菜品、收藏、评论等核心数据。
- **特点：** 成熟的关系型数据库，支持事务和复杂查询，与 Django ORM 无缝集成。

### 2.3 scikit-learn
- **作用：** 负责离线推荐算法中的特征提取与相似度计算。
- **特点：** 提供 `TfidfVectorizer`、`cosine_similarity`、`NearestNeighbors` 等工具，适合在单机上进行中小规模数据实验。

### 2.4 Spark / PySpark
- **作用：** 用于 Yelp 大规模数据集的离线统计分析与 ALS 矩阵分解模型训练。
- **特点：** 分布式批处理能力，能够处理超出单机内存的数据规模，满足毕设中"大数据组件"的展示需求。

### 2.5 前端技术栈
- **jQuery + Bootstrap 4：** 系统主要页面沿用传统管理后台模板，开发快速、组件丰富。
- **Alpine.js + HTMX：** 用于登录页等局部现代增强，实现轻量交互而不引入复杂前端工程。
- **ECharts：** 用于后台数据可视化，展示词云、热门榜单、相似度网络图等。

## 第三章 需求分析

### 3.1 系统设计目标
本系统旨在构建一个面向餐饮消费者的个性化推荐平台。系统面向**普通消费者**和**系统管理员**两类人群，核心提供中文菜品的浏览、收藏、评论以及基于用户行为的个性化推荐服务。管理员则可通过后台查看系统运行数据与可视化分析结果。系统的主要好处在于：将推荐算法的重计算前移到离线阶段，保证在线页面响应稳定；同时以工程化方式集成了协同过滤、内容推荐和矩阵分解等多种算法，形成可演示的完整闭环。

### 3.2 系统功能性需求

#### 3.2.1 普通用户角色
- **用户认证：** 注册、登录、个人信息修改、密码修改。
- **菜品浏览：** 查看菜品列表、按菜系/价格筛选、查看菜品详情（图片、描述、评分、评论）。
- **收藏与评论：** 对菜品进行收藏/取消收藏，发表评论。
- **个性化推荐：** 查看"热门推荐"榜单，查看基于个人收藏行为的 ItemCF / UserCF 推荐结果。

#### 3.2.2 管理员角色
- **后台首页：** 查看系统核心数据统计。
- **数据可视化：** 查看中文菜品词云、Yelp 评论词云、热门餐厅榜单、菜品相似度网络图等图表。
- **系统监控：** 观察推荐离线产物的生成状态与系统运行概况。

### 3.3 系统非功能性需求
- **响应速度：** 相似度计算、模型训练、词云生成等重计算通过 Django 管理命令在离线阶段完成，在线页面仅读取轻量 JSON 文件或执行简单 ORM 查询，保证页面响应速度。
- **安全性：** 采用 Django 内置的 CSRF 保护、密码哈希存储；区分普通用户与管理员的会话权限。
- **可扩展性：** 系统按 Django app 拆分（users、foods、recommendations），推荐逻辑封装为服务层，离线产物格式统一，便于后续算法替换与功能扩展。
- **界面友好性：** 沿用成熟后台模板，布局清晰；普通用户页面操作流程直观，管理员图表页面信息呈现直观。

## 第八章 总结与展望

### 8.1 总结
本文设计并实现了一个基于 Django 的餐饮个性化推荐系统。系统以中文菜品业务为核心，完成了用户注册登录、菜品浏览、收藏评论、热门推荐和基于协同过滤的个性化推荐等完整功能闭环。在算法层面，系统集成了基于收藏的 ItemCF / UserCF、基于 TF-IDF 的内容推荐，以及基于 Spark 的 ALS 矩阵分解实验。在工程层面，系统坚持"离线优先"的设计原则，将相似度计算、模型训练、词云生成等重计算前移到离线阶段，在线页面仅执行轻量查询与重排，有效保证了系统的响应速度与演示稳定性。

### 8.2 展望
尽管系统已实现了预期的功能与算法落地，但仍存在以下可优化方向：
1. **真实行为数据积累：** 当前中文菜品的协同过滤依赖演示型收藏数据，后续可引入真实长期用户行为，提升推荐的实际效果。
2. **离线评测体系：** 补充 Precision@K、Recall@K、Coverage 等评测指标，建立更客观的算法对比实验。
3. **推荐解释性：** 增加推荐原因的显式展示（如"因为您收藏了 XX，所以推荐 YY"），提升用户信任感。
4. **内容特征扩展：** 引入菜品的口味、食材、营养成分等更细粒度特征，丰富内容推荐的维度。
