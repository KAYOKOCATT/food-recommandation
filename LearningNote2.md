
# 推荐系统详解

## 推荐系统的分类与划分原则

推荐系统的划分维度众多，可以从不同角度进行分类：

| 划分维度 | 类别 | 说明 |
|---------|------|------|
| **计算时效性** | 实时推荐 / 离线推荐 | 是否基于用户实时行为即时更新推荐结果 |
| **个性化程度** | 个性化推荐 / 非个性化推荐 | 是否针对不同用户展示不同内容 |
| **算法原理** | 基于内容 / 协同过滤 / 基于知识 / 混合推荐 | 利用何种信息进行推荐决策 |
| **数据依赖** | 基于显式反馈 / 基于隐式反馈 | 是否依赖用户明确的评分或偏好表达 |

### 个性化推荐的细分类型

- **基于内容（Content-Based）**：依赖物品标签、特征工程，分析物品属性与用户历史偏好的匹配度
- **基于协同过滤（Collaborative Filtering）**：利用行为数据，包括物品协同过滤（ItemCF）和用户协同过滤（UserCF）
- **基于模型（Model-Based）**：通过机器学习模型发掘潜在规则，如矩阵分解、深度学习模型
- **基于知识（Knowledge-Based）**：依赖人工分类体系或预设规则，推荐固定的物品组合
- **基于统计与用户画像**：如"最近热门"等非个性化推荐，适用于冷启动场景

---

## 人口统计学推荐（Demographic-Based）

这是**最易实现**的推荐方法，其核心逻辑是根据系统用户的基本信息发现用户间的相关程度，然后将相似用户喜爱的其他物品推荐给当前用户。

### 用户画像构建

用户信息可以包括：
- **PGC内容**：用户首次注册时填写的标签（年龄、性别、职业等）
- **隐式数据**：登录时间、地域、设备信息等无明确语义的数据

通过聚类算法（如K-means）或预设规则，为特定标签群体推送对应物品。用户信息标签化的过程被称为**用户画像（User Profile）**。

> **用户画像定义**：企业通过收集与分析消费者的社会属性、生活习惯、消费行为等主要信息，抽象出用户的商业全貌。它是大数据应用的根基，为精准分析用户行为习惯、消费习惯提供数据基础，帮助企业快速找到精准用户群体。

## 基于内容的推荐（Content-Based）

根据推荐物品或内容的元数据，发现物品间的相关性，再基于用户过去的喜好记录，为用户推荐相似物品。

### 核心原理

通过抽取物品内在或外在的**特征值**，实现相似度计算（常用特征向量余弦相似度）。将用户个人信息的特征和物品特征相匹配，就能得到用户对物品感兴趣的程度。

$$Score(u, i) = \text{Similarity}(\text{UserProfile}(u), \text{ItemFeature}(i))$$

### 特征提取方法

**对于物品特征提取（打标签）：**
1. **专家标签（PGC Tagging）**：领域专家手动标注
2. **用户自定义标签（UGC Tagging）**：用户自主添加标签
3. **降维分析**：提取隐语义标签（LFM, Latent Factor Model）

**对于文本信息的特征提取：**
- **预处理**：分词、去除停用词
- **语义处理**：词向量表示、情感分析
- **主题模型**：潜在语义分析（LSA）、LDA主题模型

### 基于内容推荐的高层次架构

```
┌─────────────────────────────────────────────────────────────┐
│  信息源层  │  物品描述信息（文本、标签、多媒体特征）           │
├─────────────────────────────────────────────────────────────┤
│  内容分析层 │  结构化处理 → 特征提取 → 物品描述数据库         │
├─────────────────────────────────────────────────────────────┤
│  反馈收集层 │  用户行为反馈（点击、评分、停留时长）落库         │
├─────────────────────────────────────────────────────────────┤
│  特征学习层 │  反馈信息 + 物品样本 → 训练特征学习器            │
│           │  输出：用户特征向量 + 物品特征向量               │
├─────────────────────────────────────────────────────────────┤
│  推荐生成层 │  过滤组件计算相似度 → 生成推荐列表 → 排序输出     │
└─────────────────────────────────────────────────────────────┘
```

## 相似度度量方法

### 欧式距离（Euclidean Distance）

衡量两个点在多维空间中的直线距离，适用于评分数据等连续型特征。

$$d(x, y) = \sqrt{\sum_{i=1}^{n}(x_i - y_i)^2}$$

**归一化相似度**（将距离映射到[0,1]区间）：

$$\text{sim}(x, y) = \frac{1}{1 + d(x, y)}$$

### 余弦相似度（Cosine Similarity）

衡量两个向量在方向上的相似程度，忽略绝对大小，适合文本特征或评分模式比较。

$$\text{cos}(\theta) = \frac{\vec{A} \cdot \vec{B}}{||\vec{A}|| \times ||\vec{B}||} = \frac{\sum_{i=1}^{n}A_i \times B_i}{\sqrt{\sum_{i=1}^{n}A_i^2} \times \sqrt{\sum_{i=1}^{n}B_i^2}}$$

**应用场景对比：**
- **欧式距离**：关注绝对数值差异（如用户评分绝对值差异）
- **余弦相似度**：关注模式一致性（如用户评分趋势的相似性）

## 特征工程（Feature Engineering）

### 特征与目标的关系

特征是模型输入的变量，目标是模型需要预测的变量。在推荐系统中：
- **目标**：用户对物品的评分、点击率、购买意愿等
- **特征**：用户属性、物品属性、上下文信息、交叉特征等

### 特征的数量与清洗

**维度灾难与稀疏性：**
- 特征过多 → 过拟合风险、计算复杂度增加
- 特征过少 → 欠拟合，无法捕捉用户偏好

**数据清洗步骤：**
1. **缺失值处理**：填充（均值/众数）、删除、插值
2. **异常值检测**：3σ原则、箱线图法、孤立森林
3. **数据标准化**：统一量纲，消除数量级差异

### 特征的区分度与分类

**特征区分度评估：**
- 方差阈值：删除方差过小的特征（接近常数）
- 信息增益：评估特征对目标变量的信息量贡献
- 卡方检验：检验特征与目标的独立性

**特征分类：**
| 类型 | 示例 | 处理方式 |
|------|------|----------|
| 离散型特征 | 性别、类别标签 | One-Hot编码、Embedding |
| 连续型特征 | 价格、时长 | 归一化、分桶 |
| 序列特征 | 浏览历史、会话行为 | RNN、Transformer编码 |
| 交叉特征 | 用户年龄段×商品类别 | 特征交叉、FM模型 |

### 归一化与离散化

**归一化（Normalization）：**
将数据缩放到特定范围（通常是[0,1]或[-1,1]），消除不同特征量纲影响。

$$X_{norm} = \frac{X - X_{min}}{X_{max} - X_{min}}$$

**标准化（Standardization）：**
将数据转换为均值为0、方差为1的分布（Z-score标准化）。

$$Z = \frac{X - \mu}{\sigma}$$

**离散化（Discretization）：**
将连续特征划分为若干区间（分桶），降低异常值敏感度，引入非线性能力。

- **等宽分桶**：区间长度相等
- **等频分桶**：每个桶样本数相等
- **基于聚类**：利用K-means等算法分桶

## 推荐系统的常见反馈数据

### 显式反馈（Explicit Feedback）

用户明确表达的偏好，数据稀疏但置信度高：

| 反馈类型 | 示例 | 特点 |
|---------|------|------|
| 评分（Rating） | 1-5星评分、10分制 | 量化偏好强度 |
| 点赞/踩 | 喜欢、不喜欢 | 二元偏好 |
| 文字评价 | 评论、标签 | 包含丰富语义信息 |
| 收藏/关注 | 加入愿望单 | 强兴趣信号 |

### 隐式反馈（Implicit Feedback）

通过用户行为推断的偏好，数据量大但噪声高：

| 反馈类型 | 示例 | 权重设计 |
|---------|------|----------|
| 浏览行为 | 点击、曝光、停留时长 | 停留时长>点击>曝光 |
| 交互深度 | 滚动、放大、旋转 | 深度交互=高兴趣 |
| 转化行为 | 加购、购买、分享 | 购买>加购>点击 |
| 负反馈 | 快速划过、关闭、举报 | 负样本信号 |
| 时间衰减 | 历史行为vs近期行为 | 指数衰减加权 |

### 反馈数据的处理策略

**样本不均衡处理：**
- 正样本：用户明确交互的物品
- 负采样：从未交互物品中采样（ popularity-based或uniform ）
- 样本加权：根据置信度调整样本权重

**时间序列建模：**
- 最近行为赋予更高权重
- 使用滑动窗口捕获短期兴趣
- 长期偏好与短期会话分离建模

## 基于UGC的推荐（User Generated Content）

用户通过评论、标签、帖子等形式描述对物品的看法，这是联系用户和物品的重要纽带（既是行为数据也是物品内容信息）。

通常用三元组 $(u, i, b)$ 表示一条评论行为记录，表示用户 $u$ 给物品 $i$ 留下评论 $b$。评论与用户、评论与物品之间都是**多对多**关系。

### UGC推荐的挑战

1. **文本噪声**：口语化、拼写错误、情感复杂
2. **数据稀疏**：大量物品评论数极少（长尾分布）
3. **热门物品偏差**：热门物品获得更多曝光和评论
4. **时效性**：过时评论可能无法反映当前物品质量

### TF-IDF：词频-逆文档频率

常用于资讯检索和文本挖掘的加权技术。这是一种统计方法，用于评估一个字词对于一个文件或一个语料库的其中一份文件的重要程度。

**核心思想**：字词的重要性随着它在文件中出现的次数成正比增加（TF），但同时随着它在语料库中出现的频率成反比下降（IDF）。

$$\text{TF-IDF}_{i,j} = \text{TF}_{i,j} \times \text{IDF}_i$$

#### 词频 TF（Term Frequency）

指的是某一个给定的词语在该文件中出现的频率。这个数字是对词数的归一化，以防止偏向更长的文件。

$$\text{TF}_{i,j} = \frac{n_{i,j}}{n_{*,j}}$$

其中 $\text{TF}_{i,j}$ 表示词语 $i$ 在文档 $j$ 中出现的频率，$n_{i,j}$ 表示 $i$ 在 $j$ 中出现的次数，$n_{*,j}$ 表示文档 $j$ 的总词数。**TF越大，该词在文档中的重要程度越高**。

#### 逆文档频率 IDF（Inverse Document Frequency）

是一个词语普遍重要性的度量，某一特定词语的IDF，可以由总文档数目除以包含该词语之文档的数目，再将得到的商取对数得到。

$$\text{IDF}_i = \log\left(\frac{N+1}{N_i+1}\right)$$

其中 $\text{IDF}_i$ 表示词语 $i$ 在文档集中的逆文档频率，$N$ 表示文档集中的文档总数，$N_i$ 表示文档集中包含了词语 $i$ 的文档数。**分母+1是平滑处理，防止除零**。

### TF-IDF对UGC推荐的改进应用

**用户兴趣画像构建：**
1. 提取用户历史评论的所有关键词
2. 计算每个关键词在用户评论中的TF-IDF权重
3. 选取Top-K关键词作为用户兴趣标签
4. 推荐与用户兴趣标签TF-IDF向量相似的物品

**物品质量评估：**
- 提取评论中的情感关键词
- 结合TF-IDF权重和情感极性，计算物品的综合评分
- 识别特定方面的优缺点（如"电池_TFIDF高+负面情感"=续航问题）

**热门物品惩罚：**
- 对于热门物品，常用词汇的IDF值较低，降低其权重
- 突出显示包含稀有但高相关度词汇的评论

## UGC推荐与基于内容推荐的核心差异辨析

### 1. 概念本质差异

| 维度 | 基于内容推荐 (Content-Based) | UGC推荐 (User Generated Content) |
|------|---------------------------|--------------------------------|
| **定义** | 利用**物品自身的元数据**（标签、类别、描述文本）发现物品相似性 | 利用**用户产生的关于物品的内容**（评论、评分文字、用户打标签）作为推荐依据 |
| **数据主体** | 物品固有属性（PGC/专家标注、系统抓取） | 用户主动创作的行为数据（评论、帖子、标签） |
| **双重属性** | 纯物品内容特征 | **既是行为数据，又是物品内容信息**（笔记中强调） |

**关键辨析**：
- 笔记中提到基于内容依赖"**物品标签、特征工程**"，而UGC是"**行为数据且是物品内容信息**"
- UGC是用户与物品之间的**纽带**，而基于内容是物品内部的**属性描述**

### 2. 特征提取方式差异

#### 基于内容推荐
笔记中提到的特征提取：
- **专家标签**（PGC）
- **降维分析**（LFM提取隐语义标签）
- **文本特征**：分词、语义处理、情感分析、LSA

**特点**：直接分析物品的客观属性，不依赖其他用户。

#### UGC推荐
笔记中重点讨论的是**TF-IDF**在评论文本上的应用：
- 利用用户评论 $(u, i, b)$ 三元组
- 通过词频-逆文档频率衡量词语在评论集中的重要性
- 处理的是**主观评价内容**（如"电池续航很好"、"剧情拖沓"）

**特点**：利用群体智慧产生的描述性内容，包含主观情感和体验信息。

### 3. 数据关系差异

笔记中用不同数学符号表示：

**基于内容**：
- 关注物品特征向量 $\text{ItemFeature}(i)$
- 计算 $\text{Similarity}(\text{UserProfile}(u), \text{ItemFeature}(i))$
- 用户画像通过其历史交互物品的**内容特征**聚合而成

**UGC**：
- 用三元组 $(u, i, b)$ 表示，强调**行为记录**
- 评论与用户、评论与物品都是**多对多关系**
- 需要处理热门物品的惩罚（引出TF-IDF）

### 4. 应用场景差异

| 场景 | 基于内容 | UGC推荐 |
|------|---------|---------|
| **新物品冷启动** | 有效（只要有内容描述） | 困难（需要等待用户产生评论） |
| **推荐理由** | "因为你喜欢科幻电影"（标签匹配） | "看过的人都说剧情精彩"（口碑聚合） |
| **质量判断** | 可能受内容描述与实际质量脱节的影响 | 基于群体真实体验，**避免了基于内容仅依赖描述造成的质量判断干扰**（笔记中协同过滤章节提到） |

### 5. 两者的交集与融合

实际上，在现代推荐系统中，**UGC常被作为基于内容推荐的一种特征来源**：

```
基于内容推荐（广义）
├── 物品固有属性（PGC）：类别、标签、文本描述
└── 用户生成内容（UGC）：评论、评分文字、用户标签 ← 你的笔记重点
```

笔记中提到的"**用户自定义标签**"（User Tagging）正是UGC与基于内容的结合点：
- 用户给电影打标签"悬疑"，这既是UGC（用户行为），又成为物品内容特征（基于内容的输入）


### 总结对照表

| 对比项 | 基于内容 | UGC推荐 |
|--------|---------|---------|
| **数据来源** | 物品元数据、描述文本 | 用户评论、评分文字、社交帖子 |
| **信息性质** | 客观描述 | 主观评价+客观描述 |
| **核心算法** | 余弦相似度、LFM、LSA | TF-IDF、情感分析、主题模型 |
| **冷启动** | 新物品可直接提取特征 | 新物品无评论则无法使用 |
| **可解释性** | 强（标签匹配） | 中（关键词提取、口碑摘要） |
| **数据稀疏性** | 物品描述通常完整 | 评论分布极不均匀（长尾） |

**一句话概括**：基于内容推荐问的是"**这个物品是什么**"，UGC推荐问的是"**用户怎么评价这个物品**"。前者关注客观属性，后者关注主观反馈。

## 协同过滤（Collaborative Filtering, CF）

协同过滤是最经典的推荐算法之一，分为**基于近邻（Memory-Based）** 和 **基于模型（Model-Based）** 两种版本。

**基于近邻**包括：
- UserCF（基于用户的协同过滤）
- ItemCF（基于物品的协同过滤）

**基于模型**包括：
- 矩阵分解（SVD、NMF）
- 潜在语义分析（LSA）
- 概率矩阵分解（PMF）
- 深度学习模型（AutoEncoder、NeuralCF）

### 协同过滤 vs 基于内容

| 维度 | 基于内容（CB） | 协同过滤（CF） |
|------|---------------|---------------|
| 依赖数据 | 物品内容特征 | 用户-物品交互矩阵 |
| 冷启动（新物品） | 可直接推荐 | 无法推荐（无交互记录） |
| 冷启动（新用户） | 依赖用户填写的兴趣 | 依赖历史行为积累 |
| 推荐多样性 | 容易陷入相似内容 | 可跨领域推荐 |
| 可解释性 | 强（标签匹配） | 较弱（用户/物品相似） |

**协同过滤的优势：**
- **内容无关性**：不需要理解物品内容，只需用户行为数据
- **挖掘潜在兴趣**：能推荐内容差异大但用户可能喜欢的物品
- **质量筛选机制**：基于群体智慧，避免内容特征与实际质量脱节

### 基于用户的协同过滤（UserCF）

核心假设：相似用户喜欢的物品，当前用户也可能喜欢。

**算法步骤：**
1. 找到与目标用户兴趣相似的其他用户（计算用户间相似度）
2. 从这些相似用户喜欢的物品中，找出目标用户未交互过的物品
3. 根据相似度加权计算推荐分数

$$\text{Score}(u, i) = \sum_{v \in N(u)} \text{sim}(u, v) \cdot r_{v,i}$$

其中 $N(u)$ 是用户 $u$ 的相似邻居集合，$r_{v,i}$ 是用户 $v$ 对物品 $i$ 的评分。

**适用场景**：用户数量远小于物品数量的场景（如小众社区、新闻推荐）。

### 基于物品的协同过滤（ItemCF）

核心假设：用户过去喜欢的物品，与这些物品相似的物品用户也可能喜欢。

**算法步骤：**
1. 计算物品间的相似度（基于共现矩阵）
2. 对于目标用户的历史行为，找出相似的候选物品
3. 根据历史评分和物品相似度加权计算推荐分数

$$\text{Score}(u, i) = \sum_{j \in N(u)} \text{sim}(i, j) \cdot r_{u,j}$$

其中 $N(u)$ 是用户 $u$ 的历史行为物品集合。

**物品相似度计算（余弦相似度）：**

$$\text{sim}(i, j) = \frac{\sum_{u \in U} r_{u,i} \cdot r_{u,j}}{\sqrt{\sum_{u \in U} r_{u,i}^2} \cdot \sqrt{\sum_{u \in U} r_{u,j}^2}}$$

### ItemCF vs Content-Based的辨析

- **ItemCF**和**基于内容**的推荐都是基于物品相似度进行预测
- **区别**：相似度计算依据不同
  - ItemCF：从用户历史偏好行为推断（协同信息）
  - Content-Based：基于物品本身的属性特征（内容信息）

### UserCF vs ItemCF的选择策略

| 维度 | 推荐策略 | 原因 |
|------|---------|------|
| **物品数量 < 用户数量** | UserCF | 用户相似度计算量小，邻居稳定 |
| **物品数量 > 用户数量** | ItemCF | 物品相似度计算量小，可离线预计算 |
| **物品更新慢** | ItemCF | 物品相似度可离线批量计算，实时性要求低 |
| **用户兴趣稳定** | ItemCF | 历史行为权重高，推荐稳定 |
| **用户兴趣多变** | UserCF | 捕捉群体实时兴趣变化 |
| **需要可解释性** | ItemCF | "看了X的用户也看了Y"更易理解 |

**典型应用：**
- **电商、电影、音乐网站**：用户数量远大于物品数量，使用**ItemCF**
- **新闻、社交媒体**：物品（新闻/帖子）数量远超用户，且时效性强，使用**UserCF**

### 协同过滤的优缺点总结

**优点：**
- 无需领域知识，不依赖物品内容特征
- 支持发现潜在兴趣，跨品类推荐
- 社区智慧，利用集体行为数据

**缺点：**
- **冷启动问题**：新用户、新物品缺乏历史数据
- **数据稀疏性**：用户-物品矩阵极度稀疏（>99%缺失）
- **扩展性问题**：用户/物品数量增长时，计算复杂度急剧上升
- **长尾问题**：热门物品被过度推荐，冷门物品难以曝光
- **同义问题**：不同名称的相同物品被视为不同实体

## 基于模型的协同过滤

### 核心思想

用户具有一定的特征，决定着他的偏好选择；物品具有一定的特征，影响着用户是否选择它。用户之所以选择某一个商品，是因为用户特征与物品特征相互匹配。

**建模本质**：从行为数据中提取特征，给用户和物品同时打上"标签"。这与基于人口统计学的用户标签、基于内容方法的物品标签本质相同，都是特征的提取和匹配。

- **显性特征**：用户标签、物品分类标签 → 直接匹配推荐
- **隐性特征**：通过偏好数据发掘隐藏特征 → 隐语义模型（LFM）

### 基于近邻 vs 基于模型

| 维度 | 基于近邻（Memory-Based） | 基于模型（Model-Based） |
|------|-------------------------|------------------------|
| **预测方式** | 直接使用原始偏好数据，通过邻居加权 | 训练参数化模型，学习潜在规律 |
| **计算时机** | 预测时实时计算相似度 | 离线训练模型，在线快速预测 |
| **泛化能力** | 弱（记忆历史模式） | 强（学习潜在因素） |
| **稀疏处理** | 困难（依赖共现数据） | 较好（降维、正则化） |
| **类比** | K近邻分类 | 回归/分类模型 |

### 隐语义模型（Latent Factor Model, LFM）

用隐语义模型进行协同过滤的目标：
- 揭示隐藏的特征，这些特征能够解释为什么给出对应的预测评分
- 这类特征可能是无法直接用语言解释描述的，我们不需要显式定义它们，而是让模型自动学习

**降维的必要性：**
- 协同过滤依赖历史数据，但偏好数据通常是极度稀疏的
- 通过矩阵分解进行降维，将高维稀疏矩阵映射到低维隐因子空间
- 分解后的矩阵代表了用户和物品的隐藏特征（Latent Factors）

### 隐语义模型的实例

1. **pLSA**（概率潜在语义分析）：基于概率图模型的主题模型
2. **LDA**（隐式狄利克雷分布）：贝叶斯主题模型，引入先验分布
3. **矩阵因子分解模型**：
   - SVD（奇异值分解）
   - NMF（非负矩阵分解）
   - PMF（概率矩阵分解）
   - BiasSVD（带偏置的SVD）

## 矩阵因子分解（Matrix Factorization）

### 基本原理

假设用户-物品评分矩阵为 $R \in \mathbb{R}^{m \times n}$，其中 $m$ 是用户数，$n$ 是物品数。矩阵中的元素 $r_{ui}$ 表示用户 $u$ 对物品 $i$ 的评分。

**核心假设**：存在 $k$ 个隐因子（latent factors），用户的评分行为由这些隐因子决定。

将矩阵 $R$ 分解为两个低维矩阵的乘积：

$$\hat{R} = P^T \cdot Q \approx R$$

其中：
- $P \in \mathbb{R}^{k \times m}$：用户特征矩阵，$P_u$ 表示用户 $u$ 的 $k$ 维特征向量
- $Q \in \mathbb{R}^{k \times n}$：物品特征矩阵，$Q_i$ 表示物品 $i$ 的 $k$ 维特征向量

**预测评分计算：**

$$\hat{R}_{ui} = P_u^T \cdot Q_i = \sum_{k=1}^{K} p_{uk} \cdot q_{ki}$$

### 直观理解

为什么矩阵分解有效？

- **隐因子解释**：影响用户评分的因素可能是演员、题材、年代等，甚至是不具备直观语义的抽象因子
- **降维压缩**：原始矩阵 $R$ 极度稀疏，分解后的 $P$ 和 $Q$ 是稠密的低维表示
- **缺失值预测**：通过 $P_u^T \cdot Q_i$ 可以计算用户未交互物品的预测评分

### 损失函数与优化目标

我们希望预测评分矩阵 $\hat{R}$ 与原始评分矩阵 $R$ 在已知评分位置上尽可能接近。

**平方损失函数（带L2正则化防止过拟合）：**

$$\min_{P,Q} \mathcal{L} = \sum_{(u,i) \in \mathcal{K}} (r_{ui} - \hat{r}_{ui})^2 + \lambda \left( \sum_u ||P_u||^2 + \sum_i ||Q_i||^2 \right)$$

其中：
- $\mathcal{K}$：已知评分的用户-物品对集合
- $\lambda$：正则化系数，通过交叉验证确定
- 正则化项：约束用户和物品向量的模长，防止过拟合

## 模型求解算法

### 交替最小二乘法（ALS, Alternating Least Squares）

由于 $P$ 和 $Q$ 都未知且通过乘积耦合，ALS采用**坐标下降**思想：固定一个变量，优化另一个，交替进行。

**算法流程：**
1. 初始化 $Q$（通常用随机小数或全局平均值）
2. **固定 $Q$，求解 $P$**：对每个用户 $u$，求解最优 $P_u$
3. **固定 $P$，求解 $Q$**：对每个物品 $i$，求解最优 $Q_i$
4. 重复步骤2-3，直到损失函数收敛或达到最大迭代次数

#### ALS的闭式解推导

当固定 $Q$ 时，对每个用户 $u$，损失函数变为：

$$\mathcal{L}_u = \sum_{i \in I_u} (r_{ui} - P_u^T Q_i)^2 + \lambda ||P_u||^2$$

其中 $I_u$ 是用户 $u$ 评分过的物品集合。

对 $P_u$ 求导并令导数为0：

$$\frac{\partial \mathcal{L}_u}{\partial P_u} = -2\sum_{i \in I_u} (r_{ui} - P_u^T Q_i)Q_i + 2\lambda P_u = 0$$

整理得到：

$$\sum_{i \in I_u} Q_i Q_i^T P_u + \lambda P_u = \sum_{i \in I_u} r_{ui} Q_i$$

$$\left( \sum_{i \in I_u} Q_i Q_i^T + \lambda I \right) P_u = \sum_{i \in I_u} r_{ui} Q_i$$

**闭式解：**

$$P_u = \left( Q_{I_u} Q_{I_u}^T + \lambda I \right)^{-1} Q_{I_u} R_{u,I_u}^T$$

同理，固定 $P$ 时：

$$Q_i = \left( P_{U_i} P_{U_i}^T + \lambda I \right)^{-1} P_{U_i} R_{U_i,i}^T$$

其中 $Q_{I_u}$ 是用户 $u$ 交互过的物品特征矩阵，$P_{U_i}$ 是交互过物品 $i$ 的用户特征矩阵。

**ALS优点**：可以并行计算每个用户/物品的向量，适合大数据场景（Spark MLlib采用）。

### 随机梯度下降法（SGD, Stochastic Gradient Descent）

通过迭代方式更新参数，每次处理一个样本 $(u, i, r_{ui})$。

**损失函数：**

$$\mathcal{L} = (r_{ui} - P_u^T Q_i)^2 + \lambda (||P_u||^2 + ||Q_i||^2)$$

**梯度计算：**

对 $P_u$ 的梯度：
$$\frac{\partial \mathcal{L}}{\partial P_u} = -2(r_{ui} - P_u^T Q_i)Q_i + 2\lambda P_u = -2e_{ui}Q_i + 2\lambda P_u$$

对 $Q_i$ 的梯度：
$$\frac{\partial \mathcal{L}}{\partial Q_i} = -2(r_{ui} - P_u^T Q_i)P_u + 2\lambda Q_i = -2e_{ui}P_u + 2\lambda Q_i$$

其中 $e_{ui} = r_{ui} - P_u^T Q_i$ 是预测误差。

**参数更新（学习率 $\alpha$）：**

$$P_u \leftarrow P_u + \alpha (2e_{ui}Q_i - 2\lambda P_u)$$
$$Q_i \leftarrow Q_i + \alpha (2e_{ui}P_u - 2\lambda Q_i)$$

**SGD优点**：实现简单，收敛速度快；**缺点**：难以并行，对初始值敏感。

## 冷启动问题与解决方案

冷启动是推荐系统面临的核心挑战，分为三种类型：

### 1. 新用户冷启动

**问题**：新用户没有历史行为数据，无法进行个性化推荐。

**解决方案：**
- **热门推荐**：推荐全局热门物品（MostPopular）
- **个人信息引导**：注册时收集兴趣标签（新用户问卷）
- **上下文利用**：基于地理位置、设备、时间等推荐
- **试探策略（Exploration）**：
  - Epsilon-Greedy：以 $\epsilon$ 概率随机推荐，$1-\epsilon$ 概率推荐热门
  - 多臂老虎机（MAB）：UCB、Thompson Sampling平衡探索与利用
- **跨域迁移**：利用其他平台的行为数据（联邦学习）

### 2. 新物品冷启动

**问题**：新物品没有用户交互记录，协同过滤无法推荐。

**解决方案：**
- **内容-based推荐**：利用物品属性、标签、文本描述计算相似度
- **专家标注**：人工编辑打标签、写推荐语
- **曝光扶持**：在推荐流中给予新物品一定曝光机会（探索流量）
- **冷启通道**：专门的新品推荐场景，用户明确浏览新品

### 3. 新系统冷启动

**问题**：平台刚上线，用户和物品都很少。

**解决方案：**
- **专家生产内容（PGC）**：初期邀请KOL生产优质内容
- **基于内容的推荐**：不依赖历史交互，直接匹配内容相似度
- **社交关系导入**：利用用户的社交网络数据（好友推荐）
- **非个性化策略**：编辑精选、排行榜、专题运营

## 推荐系统的评估指标

### 离线评估指标

**准确性指标：**
- **RMSE**（均方根误差）：$\sqrt{\frac{1}{N}\sum(r_{ui} - \hat{r}_{ui})^2}$，衡量评分预测精度
- **MAE**（平均绝对误差）：$\frac{1}{N}\sum|r_{ui} - \hat{r}_{ui}|$，对异常值更鲁棒
- **准确率（Precision）**：Top-K推荐中用户真正喜欢的比例
- **召回率（Recall）**：用户喜欢的物品中被成功推荐的比例
- **F1-Score**：Precision和Recall的调和平均

**排序指标：**
- **AUC**（ROC曲线下面积）：衡量正样本排在负样本前面的概率
- **MAP**（平均精度均值）：考虑排序位置的准确率
- **NDCG**（归一化折损累计增益）：位置越靠前权重越高

### 在线评估方法

**A/B测试**：
- 将用户随机分为对照组（旧算法）和实验组（新算法）
- 观察核心业务指标（CTR、转化率、留存率）的差异
- 需要统计显著性检验（p-value < 0.05）

**多臂老虎机（MAB）**：
- 动态分配流量给不同算法
- 根据实时表现自动调整流量分配比例
- 在探索新算法和利用最优算法间平衡

### 多样性指标

- **个体多样性**：单个用户推荐列表中物品的多样性（基于内容相似度）
- **整体多样性**：不同用户推荐列表的差异性（Coverage）
- **长尾能力**：推荐物品中冷门物品的比例（Catalog Coverage）

## 推荐系统的架构设计

### 召回层（Retrieval）

从海量物品库（百万/亿级）中快速筛选候选集（千级）。

**召回策略：**
- **协同过滤召回**：UserCF、ItemCF、矩阵分解
- **向量召回**：利用Embedding计算相似度（FAISS、Annoy加速）
- **图谱召回**：基于知识图谱的关系路径
- **热门/运营召回**：兜底策略，保证多样性

**技术优化：**
- 倒排索引（Inverted Index）：快速查找相似物品
- 局部敏感哈希（LSH）：近似最近邻搜索，降低计算复杂度

### 粗排层（Coarse Ranking）

对召回的候选集进行初步排序（千级→百级），使用轻量级模型。

**特点：**
- 特征简单，计算速度快
- 通常使用LR、GBDT等轻量模型
- 目标：快速过滤明显不相关的物品

### 精排层（Fine Ranking）

对粗排后的物品进行精准排序（百级→十级），使用复杂深度学习模型。

**模型选择：**
- Wide&Deep、DeepFM、DIN等
- 考虑 Hundreds of features：用户画像、物品特征、上下文、交叉特征

**多目标优化：**
- 点击率（CTR）、转化率（CVR）、停留时长（Dwell Time）
- 多任务学习（MTL）：ESMM、MMoE等模型联合优化多个目标

### 重排层（Re-rank）

业务规则和用户体验优化（十级→展示级）。

**策略：**
- **多样性控制**：MMR（最大边缘相关）、DPP（行列式点过程）
- **疲劳度控制**：同一作者/类目打散，避免连续推荐相似内容
- **流量扶持**：新品、创作者扶持的插拔策略
- **上下文适配**：根据时间、地理位置调整排序

## 总结与演进趋势

### 推荐算法的演进路线

1. **传统机器学习时代**（2000-2010）：协同过滤、矩阵分解、GBDT
2. **深度学习时代**（2010-2018）：Wide&Deep、DeepFM、序列建模
3. **强化学习时代**（2018-2020）：利用用户长期反馈优化策略（延迟奖励）
4. **大模型时代**（2020-至今）：
   - 利用LLM理解内容语义（BERT、GPT-based推荐）
   - 生成式推荐（直接生成推荐序列或解释）
   - 统一多模态推荐（图文音视频统一Embedding）

### 现代推荐系统的核心挑战

- **稀疏性与冷启动**：始终存在的根本问题
- **可解释性**：深度模型的黑盒问题，需要可解释推荐（XAI）
- **公平性与偏见**：避免性别、年龄歧视，信息茧房问题
- **隐私保护**：联邦学习、差分隐私保护用户数据
- **实时性**：流式计算（Flink、Spark Streaming）支持实时特征更新
